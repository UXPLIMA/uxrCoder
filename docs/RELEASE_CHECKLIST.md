# Release Checklist

Use this checklist for publishing a new uxrCoder version.

## 1. Versioning

- [ ] Update versions in:
  - `package.json`
  - `server/package.json`
  - `vscode-extension/package.json`
- [ ] Add new entry to `CHANGELOG.md`
- [ ] Ensure README release badge/version references are updated

## 2. Quality Gates

Run from repository root:

```bash
npm --prefix server run lint
npm --prefix server test -- --run
npm --prefix server run build
npm --prefix vscode-extension run lint
npm run build
```

Optional (recommended):

```bash
npm --prefix server run profile:large-tree
```

## 3. Documentation Gates

- [ ] `README.md` (EN) updated
- [ ] `README.tr.md` (TR) updated
- [ ] Docs in `docs/` match current API/features
- [ ] `server/README.md` and `vscode-extension/README.md` updated
- [ ] `ROADMAP.md` and `CHANGELOG.md` consistent

## 4. Agent/Test System Gates

- [ ] `/agent/snapshot` and `/agent/schema/properties` working
- [ ] `/agent/command` and `/agent/commands` conflict behavior verified
- [ ] `/agent/tests/*` run/report/artifact flow verified
- [ ] Visual baseline flow tested (`assert` and `record`)

## 5. Security and Policy

- [ ] `SECURITY.md` contact/process valid
- [ ] No secrets checked into repository
- [ ] Server not exposed publicly in release instructions

## 6. Packaging

Server:
- [ ] `server/dist/` generated by build

Extension:
- [ ] `vscode-extension/out/` generated by compile
- [ ] Optional VSIX package generated and tested

## 7. Publish

- [ ] Commit release changes
- [ ] Tag release (`vX.Y.Z`)
- [ ] Push branch + tag
- [ ] Publish GitHub Release notes from `CHANGELOG.md`
- [ ] Attach artifacts if needed
